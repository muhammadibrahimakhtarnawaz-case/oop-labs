Introduction

The OOP Labs API is organised around REST. All requests must be made over HTTPS. The API returns JSON-encoded responses and uses standard HTTP response codes.

Authentication

Authenticate using an API key in the Authorization header:

curl https://api.ooplabs.io/v1/projects \ -H "Authorization: Bearer YOUR_API_KEY"

Errors

OOP Labs uses conventional HTTP response codes. Codes in the 2xx range indicate success. Codes in the 4xx range indicate a client error. Codes in the 5xx range indicate a server error.

Rate limits

The API is rate-limited to 1,000 requests per minute per API key. Exceeding this limit returns a 429 Too Many Requests response. The Retry-After header tells you when to retry.

Projects

GET /projects List all projects

Returns a paginated list of all projects in the authenticated account.

// Response { "data": [ { "id": "proj_abc", "name": "My Project", "created_at": "2024-01-01" } ], "meta": { "page": 1, "total": 1 } }
POST /projects Create a project

Creates a new project. Required fields: name.

// Request body { "name": "My New Project", "description": "Optional description" }
GET /projects/:id Get a project

Returns a single project by its ID.

PUT /projects/:id Update a project

Updates an existing project. All body fields are optional.

DELETE /projects/:id Delete a project

Permanently deletes a project and all associated resources. This action is irreversible.

Deployments

GET /projects/:id/deployments List deployments

Returns all deployments for a project, sorted by creation date descending.

POST /projects/:id/deployments Create a deployment

Triggers a new deployment. Required: environment, ref (git branch, tag, or SHA).

{ "environment": "production", "ref": "main" }

Webhooks

POST /projects/:id/webhooks Register a webhook

Registers a webhook URL for one or more events.

{ "url": "https://your-app.com/hooks/ooplabs", "events": ["deployment.succeeded", "deployment.failed"] }
