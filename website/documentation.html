Introduction

Welcome to the OOP Labs documentation. This guide will help you understand how OOP Labs works and how to integrate it into your workflow.

OOP Labs exposes a REST API with predictable URLs, standard HTTP response codes, and JSON-encoded responses. You can use our official SDKs or make raw HTTP requests.

Quickstart

Install the OOP Labs CLI to get started in under two minutes:

# Install via npm npm install -g @ooplabs/cli # Authenticate ooplabs login # Create your first project ooplabs init my-project

Or use the package SDK directly in your application:

# Install the Node.js SDK npm install @ooplabs/sdk # Python pip install ooplabs

Authentication

The OOP Labs API uses API keys for authentication. Include your key in the Authorization header:

Authorization: Bearer YOUR_API_KEY

You can generate API keys from the Settings → API Keys section of your dashboard. Keep your keys secret — treat them like passwords.

Token scopes

API keys support granular scopes:

  • read:projects — list and read project data
  • write:projects — create and update projects
  • admin — full access including billing and members

Projects

A Project is the top-level organisational unit in OOP Labs. All resources (environments, deployments, integrations) belong to a project.

// Fetch all projects (Node.js SDK) import { OopLabs } from '@ooplabs/sdk'; const client = new OopLabs({ apiKey: process.env.OOPLABS_KEY }); const projects = await client.projects.list(); console.log(projects);

Environments

Each project can have multiple Environments (e.g., production, staging, development). Environments have independent configuration, secrets, and deployment targets.

Deployments

A Deployment represents a versioned snapshot of your application deployed to an environment. Deployments are immutable once created.

GitHub Integration

Connect your GitHub repository to enable automatic deployments on every push:

  1. Go to Settings → Integrations → GitHub
  2. Click Connect GitHub and authorise the OOP Labs app
  3. Select the repositories you want to link
  4. Configure branch → environment mappings

Webhooks

OOP Labs can send HTTP POST payloads to a URL of your choice whenever specific events occur.

// Example deployment.succeeded webhook payload { "event": "deployment.succeeded", "timestamp": "2024-02-01T10:30:00Z", "data": { "deployment_id": "dep_abc123", "environment": "production", "version": "v1.4.2" } }
